1. build.gradle에서 종속할 라이브러리를 추가/수정
2. application.properties에서 환경변수 설정
3. templates : html을 작성하는 폴더
4. static : css, js, 이미지 등을 정적으로 저장하는 폴더
5. java안에 패키지폴더안에서 java프로그램을 작성
   - Config : 스프링부트의 환경 및 보안프로그램
   - Controller : 요청을 처리하는 프로그램
   - Service : 실질적인 작업을 처리하는 프로그램
   - Entity : 데이터베이스의 테이블을 설계
   - Reoisutory : 데이터베이스를 조작하는 프로그램
   - DTO : 서버와 클라이언트간의 데이터를 공유할 변수를 지정
6. test 폴더는 개발 중 메소드가 정상인지 확인하는 프로그램 작성

## 서버
웹서비스를 제공하는 컴퓨터 - 서버
스프링부트의 서버는 Tomcat을 사용
php - Apache
asp - IIS

## 클라이언트
웹서비스를 이용하는 컴퓨터 - 브라우저

## 요청(Request)
클라이언트가 서버에 신호(요청)를 보내는 것
브라우저 주소로 요청
http://localhost/sample?name="홍길동"

## 응답(Response)
서버의 요청에 따라 맞는 페이지나 데이터를 전송

## 생명주기
클라이언트가 요청을 해서 응답을 받을때까지 과정

## 호스팅(유료) - IP주로를 할당(서버의 고유주소)
서버를 구축
컴퓨터의 tomcat을 설치하고 스프링부트를 설치해서 사용할 수있는 환경

## IPv4주소는 숫자로 구성된 주소
127(국가코드).12(기업).3.12
1.1.1.1.~255.255.255.255

## 도메인 주소
ip주소를 영어나 한글주소로 변경해서 사용
12.5.4.2 ==> naver.com

## 개발자를 위한 IP주소와 도메인 주소
127.0.0.1   localhost => 서버 내부에서 서버에 접근하는 주소

## 포트
해당 주소에 접근할 수있는 출입구, 외부침입을 막기위해서 포트를 열기/닫기로 설정
1~65535
80포트 : php, asp로 개발된 웹사이트의 기본 포트(Apache)
8080포트 : java로 개발된 웹사이트의 기본 포트(Tomcat)
3306포트 : 데이터베이스
21포트 : ftp 서버 기본 포트
445, 448포트 : 이메일 서버
5000포트 : react 서버의 기본 포트

포트는 사용자가 임의로 변경이 가능, 하나의 포트는 하나의 서비스로만 사용

## 줏 지정방식
프로토콜://주소:포트
프로토콜 : 통신을 하기 위한 규칙(규약)
http : 일반페이지, https : 보안페이지 : 웹서비스를 위한 프로토콜
ftp : 파일 전송을 위한 프로토콜

http://localhost:8080
